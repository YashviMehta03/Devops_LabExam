## Developer Instructions

1. Follow proper coding standards and naming conventions.
2. Write clean, readable, and well-documented code.
3. Test your code before committing changes.
4. Use meaningful commit messages.
5. Avoid pushing directly to the main branch.
6. Handle errors and edge cases properly.
7. Keep code modular and reusable.
8. Review your code before pushing.
9. Collaborate and communicate with the team regularly.
